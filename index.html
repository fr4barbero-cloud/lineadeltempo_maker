PRIMA DI INIZIARE

Come creare e pubblicare la linea del tempo

Leggi questi passaggi prima di creare le card. Puoi riaprire le istruzioni in qualsiasi momento.

1Crea le cardPremi “Crea una nuova card” e inserisci data, titolo, immagine, testo e, se serve, un audio MP3.
2Controlla e riordinaLe card compaiono subito nell’anteprima. Trascinale nell’elenco oppure usa “Sposta prima” e “Sposta dopo”.
3Esporta il sito webPremi “Esporta la linea del tempo” e scegli una cartella vuota. L’editor creerà tutti i file del sito e anche il file progetto.timeline, necessario soltanto per riaprire il lavoro nell’editor.
4Conserva tutta la cartellaNon separare i file. La cartella completa può essere pubblicata caricandola su GitHub. Conserva la cartella così com'è: ti servirà se in futuro vorrai modificare la linea del tempo.

Esportare e pubblicare la linea del tempo

Quando hai finito, premi “Esporta la linea del tempo” e scegli una cartella vuota del computer.

L’editor preparerà automaticamente tutto ciò che serve. Dentro la cartella troverai il file principale index.html, gli altri file del sito e le cartelle con immagini, audio e testi.

Conserva sempre la cartella completa. Non rinominare, non spostare e non cancellare i file che contiene, perché devono restare collegati tra loro.

Per pubblicare il lavoro online con GitHub puoi usare uno dei due aiuti qui sotto.

▶ Guarda il video: come pubblicare su GitHub

Pubblicare la linea del tempo su GitHub

Segui questi passaggi con calma. Non devi modificare i file creati dall’editor.

  1. Vai sul sito GitHub e accedi al tuo account. Se non ne hai uno, crealo gratuitamente.
  2. Premi il pulsante “New repository” per creare un nuovo spazio online.
  3. Scrivi un nome semplice per il progetto, per esempio linea-del-tempo.
  4. Scegli l’opzione Public e premi “Create repository”.
  5. Nella nuova pagina premi “uploading an existing file” oppure “Add file” → “Upload files”.
  6. Apri sul computer la cartella creata con “Esporta la linea del tempo”.
  7. Seleziona tutti i file e tutte le cartelle contenuti al suo interno e trascinali nella pagina di GitHub.
  8. Attendi che il caricamento finisca, poi premi “Commit changes”.
  9. Apri Settings, poi Pages.
  10. In Build and deployment, scegli Deploy from a branch. Seleziona il ramo main, la cartella /(root) e premi Save.
  11. Dopo alcuni minuti GitHub mostrerà l’indirizzo del sito pubblicato. Aprilo per controllare la linea del tempo.

Importante: su GitHub devi caricare tutto il contenuto della cartella, non soltanto index.html.

  1. Apri questa pagina con Google Chrome o Microsoft Edge.
  2. Premi “Apri sito da modificare”.
  3. Seleziona la cartella completa del sito, non il solo file index.html.
  4. L’editor legge progetto.timeline e recupera immagini, audio e testi dalle cartelle collegate.
  5. Modifica, elimina, aggiungi o riordina le card.
  6. Premi nuovamente “Esporta la linea del tempo” e scegli la cartella da aggiornare.

Importante: non rinominare o spostare progetto.timeline, index.html, styles.css, app.js, data.js o le cartelle immagini, audio e testi.
